# 本地正常运行说明

## 如何正常运行并测试你改的源码

### 1. 确保 Podfile 里用了本地 MoriceLogger

已经帮你加好了，内容是：

```ruby
pod 'MoriceLogger', :path => './'
```

这样 Xcode 会用 **当前项目里的** `MoriceLogger/Classes/` 源码，你改那里的文件就会生效。

### 2. 安装 Pod

在项目根目录执行：

```bash
cd /Users/detong-morice/Documents/PackSwiftyBeaver
pod install
```

（已经执行过一次的话，改完代码一般不用再执行。）

### 3. 用 Xcode 打开工作区

**一定要打开 .xcworkspace，不要打开 .xcodeproj：**

```bash
open PackSwiftyBeaver.xcworkspace
```

或在 Finder 里双击 `PackSwiftyBeaver.xcworkspace`。

### 4. 选好目标并运行

- 在 Xcode 顶部选 **PackSwiftyBeaver** 这个 target
- 选模拟器或真机
- 按 **⌘R** 或点运行按钮

这样就是在“正常运行”你的工程，并且用的就是你改过的 `MoriceLogger/Classes/` 源码。

---

## 日常开发流程（改源码 → 正常运行）

1. 改 **`MoriceLogger/Classes/`** 里的 `.swift` 文件  
2. 在 Xcode 里 **⌘B** 编译（或直接 **⌘R** 运行）  
3. 无需再执行 `pod install`，因为 `:path => './'` 会直接用本地源码  

总结：**改 `MoriceLogger/Classes/` → 打开 `PackSwiftyBeaver.xcworkspace` → 运行 PackSwiftyBeaver**，就是你要的“正常运行”方式。
